    1. 配置git提交规范，使其符合ESlint要求
      1. git Husky相关配置
      • 安装husky，husky是一个git hook工具，可以触发git提交的各个阶段，pre-commit、commit-msg、pre-push
      • 安装完成后会自动在package.json 脚本中生成husky脚本
      • 安装完成后会自动在项目根目录生成.husky文件夹;需要手动在 .husky/pre-commit文件中配置
      1. npx cz 提交规范配置
      • 安装commitizen，commitizen是在git commit 提交meaasge的文本风格规范
      • 安装cz-conventional-changelog，初始化cz-conventional-changelog
      • 安装和初始化cz-conventional-changelog之后：
        • 自动在package.json 脚本中生成
      • 安装完成后，需要手动配置package.json commit脚本，代码提交时在终端执行npx cz 或者 npm run commmit
      1. git commit 提交规范配置
      • 安装@commitlint/config-conventional 和 @commitlint/cli
      • 项目根目录创建commitlint.config.js，配置
      • 在终端执行以下命令，使用husky生成提交信息commit-msg，验证提交信息规范
      • 上述命令执行完成后，项目根目录自动生成.husky/commit-msg文件，在commit-msg中配置
